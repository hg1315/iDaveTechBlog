[agents.md](https://agents.md/)에서 제안하는 **AGENTS.md**는 저장소 루트(또는 하위 패키지)에 두는 표준 마크다운 파일로, 사람용 README를 보완해 **코딩 에이전트**가 빌드·테스트·관례를 빠르게 파악하도록 돕는 역할을 합니다.

## README와 무엇이 다른가

| 구분 | 역할 |
| --- | --- |
| **README.md** | 사람용 빠른 시작, 프로젝트 소개, 기여 가이드 |
| **AGENTS.md** | 에이전트용 추가 맥락 — 빌드 단계, 테스트, 저장소 관례 등 README에 넣기 부담스럽거나 사람 기여자에게는 덜 중요한 내용 |

분리하는 이유는 다음과 같습니다.

- 에이전트에게 **일관된 위치**에서 지침을 주기 위함
- README를 **간결하게** 유지
- README·기존 문서와 **보완 관계**로 정밀한 에이전트 지침을 제공

별도 상표 형식이 아니라 **누구나 채택할 수 있는 이름과 형식**으로 두었다는 점이 agents.md에서 강조합니다.

## 여러 에이전트에서 함께 쓰기

하나의 AGENTS.md 정의는 GitHub Copilot Coding agent, UiPath Autopilot 등 **여러 코딩 에이전트·도구와 호환**되도록 설계되었다고 안내합니다. 생태계가 커지면서 같은 파일을 재사용하기 쉽게 한 선택입니다.

## 작성 방법 요약

1. **저장소 루트에 `AGENTS.md` 추가** — 많은 에이전트는 요청 시 초안 생성도 지원합니다.
2. **프로젝트에 맞게 채우기** — 예시로 자주 쓰이는 항목:
   - 프로젝트 개요
   - 빌드·테스트 명령
   - 코드 스타일
   - 테스트 지침
   - 보안 고려 사항
3. **그 외 팀 온보딩에 맞는 내용** — 커밋·PR 규칙, 배포 단계, 큰 데이터셋 위치, 보안 유의사항 등 동료에게 입으로 전할 내용을 그대로 적어도 됩니다.
4. **대형 모노레포** — 패키지마다 **중첩 `AGENTS.md`**를 둘 수 있습니다. 에이전트는 디렉터리 트리에서 **가장 가까운 파일**을 읽으며, 가까운 쪽이 우선합니다. (OpenAI 저장소 등 다수 파일 운용 사례가 언급됩니다.)

## FAQ 요약

- **필수 필드는 없음** — 일반 마크다운이며, 원하는 제목 구조로 작성하면 됩니다.
- **지침이 충돌하면** — 편집 중인 파일에 **가장 가까운 AGENTS.md**가 우선하고, **사용자 채팅의 명시적 지시**가 그보다 위입니다.
- **테스트 명령을 적으면** — 에이전트가 관련 검증을 실행하고 실패를 고치려 시도할 수 있습니다.
- **언제든 수정** — 살아 있는 문서로 유지하면 됩니다.

### 기존 파일 이름 바꾸기

`AGENT.md` 등을 쓰던 경우 예시처럼 이름을 바꾼 뒤 **심볼릭 링크**로 이전 이름을 유지할 수 있습니다.

```bash
mv AGENT.md AGENTS.md && ln -s AGENTS.md AGENT.md
```

### 도구별 참고 설정

- **Aider**: `.aider.conf.yml`에 `read: AGENTS.md`
- **Gemini CLI**: `.gemini/settings.json`에 `"context": { "fileName": "AGENTS.md" }` 등

## 배경과 거버넌스

AGENTS.md는 Codex, Amp, Jules, Cursor, Factory 등 **에이전틱 개발 생태계 협업**에서 나왔으며, 어떤 코딩 에이전트를 쓰든 공개 형식으로 유지·발전시키자는 취지라고 합니다. 현재는 **Linux Foundation 산하 Agentic AI Foundation([aaif.io](https://aaif.io/))**에서 관리합니다. 자세한 소식은 [OpenAI 발표](https://openai.com/index/agentic-ai-foundation/) 등을 참고하면 됩니다.

## 이 프로젝트의 AGENTS.md

이 블로그 메이커 저장소의 `AGENTS.md`에는 Next.js 관련 **에이전트 규칙**(버전별 API·관례 차이, `node_modules/next/dist/docs/` 가이드 참고 등)이 짧게 들어 있습니다. 범용 가이드와 마찬가지로, **프레임워크별 주의사항**을 에이전트 전용 파일에 모아 두는 좋은 예입니다.
